## 7 — RPC `crear_venta` (venta atómica) `d36d950`

> Esta es la prueba más importante: la venta ahora es **transaccional** (todo o nada).

**7.A — Venta normal:**
1. Tené un producto con **stock 1**.
2. POS → Nueva venta → agregá ese producto, cantidad **1** → confirmá la venta.

**Verificación en Supabase** (reemplazá `<NV>` por el `numero_venta` que generó):
```sql
-- 1) Comprobante
select id, numero_venta, fecha, total, estado_pago
from comprobantes where empresa_id = '<EMP>' order by fecha desc limit 1;

-- 2) Items (usá el id del comprobante de arriba como <COMP_ID>)
select * from comprobante_items where comprobante_id = '<COMP_ID>';

-- 3) Caja
select concepto, monto, fecha from movimientos_caja where concepto ilike '%<NV>%';

-- 4) Inventario (el stock debe haber bajado a 0)
select motivo, tipo, cantidad, fecha from movimientos_inventario where motivo ilike '%<NV>%';
select stock_actual from productos where id = '<PRODUCTO_ID>';
```
✅ Verde si: existen las 4 filas (comprobante + item + caja + inventario), con **mismo timestamp** y datos coherentes, y `stock_actual = 0`.

**7.B — Forzar fallo (rollback):**
1. Tomá un producto con **stock 0** (o vendé hasta dejarlo en 0).
2. Intentá venderlo.

✅ Verde si: muestra **error** y **NO se crea** ningún comprobante nuevo. Verificalo:
```sql
select count(*) from comprobantes
where empresa_id = '<EMP>' and fecha > now() - interval '2 minutes';
```
El conteo no debe haber aumentado por el intento fallido.
